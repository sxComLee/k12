# 项目级 AGENTS

## 当前目录用途
本目录用于将甲种本教材 PDF 转为适合学习与浏览的结构化 HTML 内容。

当前阶段先以 `代数第一册(甲种本)` 作为样板，验证：

- 章节与小节结构提取
- 教材正文到 HTML 的排版策略
- 面向学习场景的页面组织方式
- 可复用到其他教材的模板与流程

## 文件分类
- `_sandbox/`：原始 PDF、OCR 文本、中间抽取结果、临时实验产物
- `books/<book-slug>/`：单本教材的结构化内容、页面资源与最终 HTML
- `shared/`：跨教材复用的样式、模板、脚本与组件
- `docs/`：设计文档、计划文档、约束说明

## 命名规则
- 目录、HTML 文件、资源文件统一使用小写英文或拼音 slug，单词之间用 `-` 连接
- 单本教材目录命名为 `books/<book-slug>/`
- 章节页面命名优先使用 `chapter-xx.html`、`section-x-y.html`
- 结构化数据优先使用 `json` 或 `md`，名称与页面 slug 对齐

## 临时产物位置
- 一次性脚本输出、OCR 结果、调试截图、抽取缓存统一放入 `_sandbox/`
- 未确认会长期保留的文件，不进入 `books/`、`shared/` 或 `docs/`

## 清理与归档规则
- `_sandbox/` 中验证完成且可复现的中间产物可删除，不作为正式交付物
- 正式保留的模板、样式、脚本需从 `_sandbox/` 提升到 `shared/`
- 正式保留的教材内容与页面需落入对应的 `books/<book-slug>/`

## 当前默认约束
- 当前只处理 `代数第一册(甲种本)` 作为样板
- 不直接覆盖原始 PDF
- 先定义结构化与学习展示方案，再进入批量生成
