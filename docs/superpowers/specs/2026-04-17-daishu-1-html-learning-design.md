# 代数第一册（甲种本）HTML 学习版设计文档

## 1. 背景

当前目标是将 `代数第一册（甲种本）` PDF 整理为适合在线学习的 HTML 版本。

这个版本不是简单的 PDF 转网页，而是要在不改变教材数学结论与原文主线的前提下，通过更清晰的结构组织、视觉层级和必要的交互效果，提升理解效率与学习体验。

当前阶段只做单本样板，验证方法可行后再迁移到其他甲种本教材。

## 2. 当前样板范围

- 样板教材：`代数第一册（甲种本）`
- 工作目标：完成一套可复用的“教材 PDF -> 结构化内容 -> HTML 学习页”样板流程
- 当前不做：批量覆盖全部教材、数据库系统、复杂后台平台化建设

## 3. 目标与非目标

### 3.1 目标

- 保留教材原文主线与数学严谨性
- 将内容组织为清晰的 `书 -> 章 -> 节` 学习结构
- 为每节增加学习辅助层，帮助读者更快理解
- 加入少量但有效的动态、交互式效果
- 形成可迁移到其他教材的模板、样式与生成流程

### 3.2 非目标

- 不改写教材中的数学结论
- 不把教材重构为完全脱离原文的“视频课 / 任务流课程”
- 不为了现代感加入无意义动效
- 不在样板阶段优先建设复杂 CMS、账号系统、数据库后台

## 4. 设计原则

### 4.1 内容原则

- 原文为准：教材原文是唯一权威内容来源
- 辅助分层：新增说明、图示、交互只能作为辅助层，不能替代原文
- 顺序不乱：章、节、正文主线顺序不改
- 可追溯：每个原文块保留来源页码，便于回查 PDF

### 4.2 体验原则

- 清晰优先于花哨
- 严谨优先于炫技
- 交互必须降低理解成本，否则不做
- 视觉上必须明确区分“原文层”和“辅助层”

### 4.3 技术原则

- 先做静态站点生成，不先做平台
- 先做结构化数据，不直接手写整站 HTML
- 轻量优先，不默认引入大型前端框架
- 自动化优先，但允许局部人工校正

## 5. 方案比较与决策

### 5.1 备选方案

#### 方案 A：忠实重排型

只做目录、层级排版、折叠展开、跳转导航。

- 优点：最稳、最接近原书
- 缺点：对“更直观、更便于理解”的提升有限

#### 方案 B：双层学习型

保留原文主线，同时增加学习辅助层，如本节目标、关键定义卡片、例题分步展开、练习提示显隐、概念关系图、函数图像交互。

- 优点：兼顾严谨性与学习效率
- 缺点：必须严格控制辅助层边界

#### 方案 C：课程化重构型

将教材改造成强交互课程页，原文被重新拆成卡片与任务流。

- 优点：最直观
- 缺点：最容易偏离教材原貌，样板阶段风险过高

### 5.2 当前决策

采用 **方案 B：双层学习型**。

这是当前约束下的最佳方案：既保持教材的严谨主线，又能通过结构化与交互增强理解效率，并且可复用到下一本教材。

## 6. 信息架构

整体层级采用：

- 书籍首页
- 章节页
- 小节页

### 6.1 书籍首页

书籍首页承担以下职责：

- 说明教材定位与学习方式
- 展示章节总览
- 明确这是基于甲种本原文整理的学习版
- 引导用户进入各章

### 6.2 章节页

章节页承担以下职责：

- 说明本章主题与知识边界
- 展示本章的节次结构
- 提供章内知识地图
- 区分“节内容”和“章级小结 / 复习参考题”

### 6.3 小节页

小节页是主要学习页面，承担以下职责：

- 承载原文主线
- 插入学习辅助层
- 承担本节导航、交互与复习

## 7. 小节页标准结构

每个小节页统一采用以下结构：

1. `页头区`
   - 面包屑
   - 章 / 节标题
   - 原书页码范围
   - 返回上层入口

2. `学习导航区`
   - 本节目标
   - 前置知识
   - 推荐学习顺序

3. `原文主线区`
   - 按原文顺序呈现定义、性质、例题、练习、图示、注记等

4. `辅助理解区`
   - 关键概念卡片
   - 概念关系图
   - 动态演示
   - 例题分步讲解

5. `练习区`
   - 课堂练习 / 习题分组展示
   - 提示或答案默认折叠

6. `页尾区`
   - 本节小结
   - 上下节跳转
   - 返回本章

## 8. 内容块模型

目标不是把每节塞成一个长页面，而是将内容拆分为可识别、可复用、可交互的块。

### 8.1 原文块

原文块是教材权威内容，必须保留原意与顺序。

- `section-intro`：本节导入
- `definition`：定义、概念、记号约定
- `property`：性质、结论、公式、法则
- `example`：例题与解答
- `exercise`：练习、习题、复习题
- `figure`：图、表、示意结构
- `note`：脚注、注释、补充说明
- `summary`：小结

### 8.2 辅助块

辅助块只为理解服务，必须依附对应原文块存在。

- `learning-goals`：本节目标
- `key-points`：关键概念卡片
- `concept-map`：概念关系图
- `step-by-step`：例题分步展开
- `interactive-demo`：函数图像 / 几何 / 关系交互
- `exercise-hint`：练习提示与答案显隐
- `common-mistakes`：易错点

### 8.3 组织规则

- 每个原文块保留来源页码范围
- 辅助块按“贴着原文走”的原则插入
- 例题默认展示原题与原解，分步讲解作为可展开层
- 练习默认展示题目，提示与答案折叠
- 原图优先保留其语义；交互图仅作辅助，不替代原图

## 9. 内容抽取与切块规则

### 9.1 结构识别对象

从 PDF 文本中优先识别以下结构：

- `第一章`、`第二章` 等章标题
- `一`、`二`、`三` 等章内大段落标题
- `1.1`、`1.2` 等小节标题
- `例 1`、`例 2`
- `练习`
- `习题 一`、`习题 二`
- `小结`
- `复习参考题`

### 9.2 切块原则

- 目录仅作为辅助，不作为唯一真源
- 真正结构以正文标题与内容标识为准
- 连续正文按段落聚合，避免切得过碎
- 公式、集合表示、推导链条在必要时单独成块
- 自动识别失败时允许人工校正

## 10. 交互设计边界

交互只服务理解，不服务装饰。

### 10.1 结构型交互（默认必做）

- 章节树导航
- 当前节高亮
- 上一节 / 下一节跳转
- 原文块折叠与展开
- 练习提示 / 答案显隐
- 例题步骤逐步展开
- 页内锚点跳转

### 10.2 理解型交互（默认必做）

- 概念卡片点击展开
- 概念关系图联动高亮
- 原文与辅助解释互相定位
- 集合图 / 关系图的状态切换

### 10.3 数学型交互（按内容启用）

- 函数图像参数滑块
- 单调性、奇偶性动态图示
- 反函数关于 `y = x` 对称的交互演示
- 映射 / 一一映射 / 逆映射箭头关系图
- 数轴区间、集合运算动态演示

### 10.4 明确不做

- 无意义入场动画
- 自动播放复杂动效
- 打断阅读节奏的悬浮装饰组件
- 过度卡片化导致原文主线断裂

## 11. 技术与产物设计

### 11.1 技术路线

采用静态站点生成流程：

`PDF 抽取 -> 文本规范化 -> 结构识别 -> 内容切块 -> 辅助层配置 -> 页面渲染 -> 前端交互挂载`

当前不引入数据库与后台系统。

### 11.2 数据来源判断

`代数第一册（甲种本）` 可以通过 `PDFKit` 直接提取目录、正文与页码信息，因此当前样板不需要先走 OCR 作为主流程。

OCR 可作为后续其他教材的兜底方案，而不是本样板主流程。

### 11.3 结构化内容格式

结构化内容以 `JSON` 为主，必要时允许少量 `Markdown + frontmatter`。

原因：

- 比直接拼 HTML 更稳
- 便于记录块类型、页码、顺序与交互配置
- 便于后续批量迁移到其他教材

### 11.4 模板渲染策略

使用统一模板渲染以下页面：

- 书籍首页模板
- 章节页模板
- 小节页模板

样式、模板、交互脚本都放在可复用的 `shared/` 下，不和单本教材内容混杂。

## 12. 文件与目录结构

建议目录如下：

```text
_sandbox/
  daishu-1/
    raw/
    normalized/
    debug/

books/
  daishu-1/
    content/
    site/

shared/
  templates/
  styles/
  scripts/

docs/
  superpowers/
    specs/
```

### 12.1 目录职责

- `_sandbox/daishu-1/`
  用于 PDF 抽取文本、中间调试产物、人工校正草稿

- `books/daishu-1/content/`
  存放结构化教材内容

- `books/daishu-1/site/`
  存放最终生成的 HTML、CSS、JS 与资源

- `shared/templates/`
  存放跨教材复用模板

- `shared/styles/`
  存放跨教材复用样式

- `shared/scripts/`
  存放解析、生成、交互脚本

## 13. 样板阶段最小交付

样板阶段的最低交付为：

- `1 本教材样板`
- `1 个书籍首页`
- `若干章节页`
- `至少 1 个完整章节下的全部小节页生成`
- `至少 3 类有效交互`

建议优先实现的三类交互：

- 集合关系图交互
- 函数图像参数演示
- 练习提示 / 答案显隐

## 14. 风险与处理策略

### 14.1 目录抽取不稳定

- 处理：目录只作辅助，正文标题识别为准

### 14.2 公式或符号断裂

- 处理：样板阶段允许局部人工校正，不追求第一次全自动

### 14.3 原图难以直接复用

- 处理：先保留原图语义占位，优先补做关键交互图

### 14.4 辅助层压住原文主线

- 处理：强制区分视觉层级，并支持收起辅助层

## 15. 验收标准

样板完成后，应满足以下标准：

- 读者可以从书籍首页顺畅进入章、节
- 每节页面层级清晰，原文可追溯到页码
- 定义、性质、例题、练习能被明确区分
- 交互效果确实帮助理解，而不是纯装饰
- 生成流程可迁移到下一本教材，而不是一次性工程

## 16. 当前结论

当前样板项目的核心方向已经明确：

- 以 `代数第一册（甲种本）` 为单本样板
- 采用“原文主线层 + 学习辅助层”的双层学习型方案
- 用静态站点生成流程完成结构化与渲染
- 先把单本样板做对，再复制到其他教材

下一步不直接进入编码实现细节，而是基于本设计文档继续写实施计划。
